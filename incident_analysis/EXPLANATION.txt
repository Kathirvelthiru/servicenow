================================================================================
INCIDENT PATTERN ANALYSIS SYSTEM - FILE EXPLANATIONS
================================================================================

This document explains two complementary Python scripts that form an incident
pattern discovery and classification system for IT Service Management (ITSM).

================================================================================
1. updated_finder_4500.py - PATTERN DISCOVERY ENGINE
================================================================================

PURPOSE:
--------
Automatically discovers recurring incident patterns from a large dataset of
4,500 incidents using an LLM (Llama 3.1).

HOW IT WORKS:
-------------

Step 1: LOAD DATA
  - Reads Excel file: dataset_4500_similar.xlsx
  - Converts data to a list of incident dictionaries
  - Total incidents loaded: ~4,500

Step 2: CHUNK & PROCESS
  - Divides incidents into chunks of 50
  - For each chunk, sends incident details to the LLM
  - Prompt asks: "What are the recurring patterns in these incidents?"

Step 3: EXTRACT PATTERNS
  - LLM outputs pattern descriptions as JSON objects
  - Each pattern has:
    * pattern_name: Name of the incident type
    * description: What this pattern represents
    * rules: Classification rules (e.g., "Title contains X", "Category is Y")

Step 4: DEDUPLICATE & LIMIT
  - Removes duplicate patterns (same name, different cases)
  - Keeps only the top 15 patterns

Step 5: SAVE OUTPUT
  - Writes discovered patterns to: incident_patterns_top_15.json
  - This file becomes the library for the matcher script

KEY CONFIGURATION:
------------------
DATASET_FILE = "dataset_4500_similar.xlsx"
OUTPUT_FILE = "incident_patterns_top_15.json"
MODEL_NAME = "llama3.1:latest"
CHUNK_SIZE = 50
TOP_PATTERN_COUNT = 15
RETRY_LIMIT = 2

KEY FUNCTIONS:
--------------
- extract_patterns(text: str) → List[Dict]
  Robustly parses JSON from LLM output, handles markdown wrapping

- load_data(state) → None
  Loads Excel file into memory

- generate_patterns(state) → None
  Main processing loop - chunks data, calls LLM, extracts patterns

- save_output(state) → None
  Writes final patterns to JSON file

EXECUTION:
----------
python updated_finder_4500.py

OUTPUT:
-------
incident_patterns_top_15.json - JSON array of 15 incident patterns

EXAMPLE OUTPUT STRUCTURE:
[
  {
    "pattern_name": "Database Connection Timeout",
    "description": "Users unable to connect to database during peak hours",
    "rules": [
      "Title contains 'timeout' or 'connection'",
      "Category is 'Database'",
      "Description contains 'peak hours'"
    ]
  },
  ...
]

================================================================================
2. incident_pattern_matcher_02.py - PATTERN MATCHER/CLASSIFIER
================================================================================

PURPOSE:
--------
Classifies new/incoming incidents against the 15 patterns discovered by the
first script. Runs interactively to match user-provided incidents.

HOW IT WORKS:
-------------

Step 1: LOAD PATTERNS
  - Reads the incident_patterns_top_15.json file
  - Loads all 15 predefined patterns into memory

Step 2: GET USER INPUT
  - Prompts user to enter an incident with 4 fields:
    * Title: Short incident summary
    * Description: Detailed explanation
    * Category: Main incident category
    * Sub Category: Specific subcategory

Step 3: MATCH WITH LLM
  - Sends both the new incident AND all 15 patterns to the LLM
  - Prompt asks: "Which single pattern best matches this incident?"
  - LLM analyzes semantic meaning (not just keywords)
  - Returns:
    * matched_pattern: Name of the best matching pattern
    * confidence: High, Medium, or Low
    * reason: Justification for the match

Step 4: DISPLAY RESULT
  - Prints the match result with confidence and reasoning

KEY CONFIGURATION:
------------------
PATTERN_FILE = "incident_patterns_top_15.json"
MODEL_NAME = "llama3.1"
temperature=0.0 (deterministic - same input always gives same output)
base_url="http://localhost:11434" (Local Ollama server)

KEY FUNCTIONS:
--------------
- load_patterns() → List[Dict]
  Reads and returns patterns from JSON file

- extract_json(text: str) → Dict
  Parses JSON object from LLM response using regex

- match_incident_with_llm(incident, patterns) → Dict
  Core matching function:
  1. Formats patterns as JSON string
  2. Builds classification prompt
  3. Calls LLM
  4. Extracts and returns JSON result

EXECUTION:
----------
python incident_pattern_matcher_02.py

INTERACTIVE INPUT EXAMPLE:
--------------------------
Enter Incident Details
Title: Database connection timeout
Description: Users unable to log in during peak hours
Category: Database
Sub Category: Performance

OUTPUT:
-------
{
  "matched_pattern": "Database Connection Timeout",
  "confidence": "High",
  "reason": "Incident involves database connectivity issues during high load"
}

================================================================================
SYSTEM WORKFLOW - HOW THEY WORK TOGETHER
================================================================================

     [Raw Dataset: 4,500 Incidents]
               ↓
     [updated_finder_4500.py]
     (Discovery Engine)
               ↓
     [incident_patterns_top_15.json]
     (Pattern Library)
               ↓
     [incident_pattern_matcher_02.py]
     (Classifier)
               ↓
     [New Incident] → [Matched Pattern + Confidence]

WORKFLOW EXPLANATION:
---------------------
1. Run updated_finder_4500.py ONCE to generate a library of 15 patterns
2. Run incident_pattern_matcher_02.py MANY TIMES to classify new incidents
3. Each time a new incident comes in, the matcher quickly identifies which
   of the 15 known patterns it belongs to

================================================================================
COMPARISON TABLE
================================================================================

Feature                 | updated_finder_4500.py      | incident_pattern_matcher_02.py
------------------------|-----------------------------|---------------------------------
Purpose                 | Pattern Discovery           | Pattern Classification
Input                   | 4,500 incidents (batch)     | 1 new incident (interactive)
Type                    | Batch Processing            | Interactive
LLM Task                | Extract patterns            | Match against patterns
Output                  | incident_patterns_top_15.json | Console match result
Runs                    | Once (setup)                | Many times (ongoing use)
Temperature             | 0.1 (creative)              | 0.0 (deterministic)
Execution Time          | ~15-30 minutes              | ~5-10 seconds per incident

================================================================================
TECHNICAL DETAILS
================================================================================

DEPENDENCIES:
- pandas: Data manipulation (reading Excel)
- langchain_ollama: LLM interface
- langgraph: Graph-based workflow (only in finder)
- json, re: JSON parsing and regex
- typing: Type hints

LLM REQUIREMENTS:
- Ollama installed locally
- llama3.1 model downloaded
- Server running on http://localhost:11434

JSON PARSING:
Both scripts use robust JSON extraction to handle LLM output variations:
- Strips markdown code blocks (```json ... ```)
- Finds JSON within text
- Falls back to regex matching individual objects

================================================================================
USE CASES
================================================================================

1. INITIAL SETUP:
   Run updated_finder_4500.py to analyze historical incidents and create
   a pattern library. This is a one-time setup step.

2. ONGOING CLASSIFICATION:
   As new incidents arrive, use incident_pattern_matcher_02.py to:
   - Quickly identify incident patterns
   - Prioritize response based on pattern severity
   - Route to appropriate teams
   - Enable better incident management and prevention

3. PATTERN UPDATES:
   Periodically rerun updated_finder_4500.py with new incident data to
   discover emerging patterns and update the library.
